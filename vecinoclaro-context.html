Capítulo 01
Resumen Ejecutivo

VecinoClaro es una plataforma SaaS (Software as a Service) bimonetaria diseñada específicamente para la gestión de condominios y residencias en Venezuela. Nace de una necesidad real: los administradores de condominios venezolanos operan con herramientas inadecuadas —planillas de Excel perdidas en grupos de WhatsApp, comprobantes de pago en carpetas sueltas, tasas de cambio que cambian diariamente según el BCV, y una constante sensación de caos financiero que erosiona la confianza entre vecinos.

La plataforma resuelve este problema combinando contabilidad bimonetaria USD/VES en paralelo, tasa BCV automática diaria, comprobantes de pago verificados con Inteligencia Artificial, y un libro contable inmutable con cadena SHA-256 que garantiza transparencia auditable. Todo ello adaptado a la realidad venezolana: Pago Móvil, Zelle, transferencias bancarias locales, y conformidad con la Ley de Propiedad Horizontal.

El nombre VecinoClaro evoca el eslogan "Cuentas Claras, Vecinos Claros" —un juego de palabras que transmite que cuando las cuentas están claras (transparencia financiera), las relaciones entre vecinos también lo están. La plataforma no solo gestiona dinero; restaura la confianza comunitaria.

▸ Problema que resuelve
En Venezuela, un condominio típico de 12 viviendas pierde unas 6 horas semanales en tareas administrativas manuales: conciliar comprobantes por WhatsApp, calcular conversiones USD/VES con tasas desactualizadas, llevar morosidad en Excel, y responder preguntas repetitivas de residentes. Proveer herramientas tecnológicas accesibles, bimonetarias y auditables que eliminen el caos administrativo de los condominios venezolanos. Cada feature que construimos debe reducir fricción, aumentar transparencia, o restaurar confianza entre vecinos y administradores.

Capítulo 03
Plataforma y Arquitectura

VecinoClaro es una aplicación web single-page (SPA) construida sobre Next.js 16 con App Router, que se ejecuta completamente en una sola ruta (/) con navegación basada en estado (Zustand) en lugar de rutas URL tradicionales. Esto permite una experiencia fluida tipo app móvil desde el primer load, sin recargas de página.

La arquitectura es multi-tenant: cada condominio es un tenant aislado. Un usuario puede pertenecer a múltiples condominios con distintos roles (admin en uno, residente en otro). El aislamiento de datos se garantiza a nivel de query Prisma (condominiumId en cada consulta).

▸ Decisiones arquitectónicas clave
Ruta única /: toda la app vive en una sola URL. La navegación entre vistas (dashboard, pagos, viviendas, etc.) se maneja con estado Zustand, no con rutas. Esto da una experiencia tipo app móvil instantánea.

SQLite para desarrollo, PostgreSQL para producción: SQLite permite iteración rápida local. Para deploy en Vercel/Cloudflare se migrará a Turso (SQLite distribuido) o Neon (PostgreSQL serverless).

API Routes de Next.js: 63 endpoints REST que viven en src/app/api/. Cada uno usa getServerContext() para extraer usuario + condominio + rol de la sesión JWT.

Features destacadas que diferencian a VecinoClaro:

  • Contabilidad bimonetaria con snapshot de tasa BCV: cada transacción se guarda con el valor en USD, el valor en VES, y la tasa BCV exacta del día. Auditoría criptográfica completa.
  • Comprobantes con IA Groq (Llama 3.2 90B Vision): cuando un residente sube un comprobante de pago, la IA hace OCR + detección de fraude. 4 API keys con rotación automática para evitar rate limits.
  • Códigos únicos por vivienda: cada vivienda tiene un código (ej: 3A-RSRZ) que se usa como referencia en los pagos, permitiendo conciliación automática.
  • Hash chain SHA-256 inmutable: cada asiento contable sella el anterior con su hash. La cadena no se puede alterar retroactivamente sin romperse. Transparencia nivel blockchain sin blockchain.
  • Votaciones con peso por indiviso: conforme a la Ley de Propiedad Horizontal de Venezuela, cada voto vale según la alícuota de la vivienda, no "una persona = un voto".
  • Conciliación bancaria CSV: el admin sube el estado de cuenta del banco en CSV y el sistema cruza automáticamente con los pagos registrados.
  • Generación batch de facturas mensuales: con un click se generan las facturas de todos los residentes para el período, aplicando mora/intereses automáticamente.
  • Notificaciones in-app + email: 4 flujos automáticos (pago aprobado/rechazado, nuevo aviso, nueva votación) disparan notificación en la campana + email con template HTML.
Capítulo 05
El Sistema Bimonetario

El corazón diferenciador de VecinoClaro es su motor bimonetario. En Venezuela, la economía opera simultáneamente en USD (dólares) y VES (bolívares), con una tasa de cambio que el Banco Central de Venezuela (BCV) publica diariamente. Cualquier software de gestión que ignore esta dualidad es inútil en el mercado venezolano.

VecinoClaro trata cada monto como un par (USD, VES, bcvRateId) indivisible. Cuando se registra un pago de USD 45.00, el sistema automáticamente:

  • Consulta la tasa BCV del día (ej: 622,21 Bs/USD vía DolarApi.com)
  • Calcula el equivalente en VES (Bs 27.990,00)
  • Guarda los 3 valores (USD, VES, ID de la tasa) en la base de datos
  • Crea un asiento en el ledger con hash SHA-256 que incluye los 3 valores
  • Si la tasa cambia mañana, el historial mantiene el valor original al día de la transacción
▸ Por qué esto importa
Sin snapshot de tasa, un pago de USD 45 hecho en enero (cuando la tasa era 480 Bs/USD) aparecería hoy (con tasa 622) como si fueran Bs 27.990 cuando en realidad fueron Bs 21.600. Esta ambigüedad destruye la confianza contable. VecinoClaro elimina este problema al guardar siempre el trío (USD, VES, tasa) atómico.

La tasa BCV se sincroniza automáticamente cada día (con cron job planeado para Fase 12). Si la API de DolarApi.com falla, hay un fallback de 3 niveles: (1) reintentar API, (2) usar última tasa conocida, (3) permitir input manual por el admin con validación. El widget de tasa en el topbar muestra "Actualizado" (verde) o "Desactualizado" (ámbar) según si la tasa es de hoy.

Capítulo 06
Seguridad, IA y Auditoría
🔐 Cadena SHA-256

Cada asiento del libro contable incluye el hash del asiento anterior, formando una cadena inmutable. Alterar un asiento retroactivamente rompe toda la cadena posterior. Permite auditoría criptográfica sin blockchain.

🤖 IA Groq Llama 3.2 90B Vision

Modelo vision-language que analiza cada comprobante subido: extrae monto, fecha, banco, número de referencia; y detecta señales de fraude (imagen manipulada, montos inconsistentes, bancos falsos). 4 API keys con rotación automática.

Flujo completo de un comprobante con IA:

  • Residente sube foto del comprobante de Pago Móvil/transferencia
  • El sistema calcula un hash SHA-256 de la imagen (anti-duplicados)
  • Groq Llama 3.2 90B Vision analiza: OCR de monto, fecha, banco, referencia + detección de fraude
  • El admin ve TODOS los pasos de la IA en un panel de transparencia: qué detectó, con qué confianza, qué señales de fraude encontró
  • El admin toma la decisión final: aprobar o rechazar (con razón)
  • Al aprobar, se crea automáticamente el Payment + asiento contable en el ledger
  • El residente recibe notificación in-app + email del resultado

Capítulo 08
PWA y Visión Móvil

VecinoClaro está diseñado desde el inicio como una Progressive Web App (PWA). Esto significa que la misma aplicación web puede instalarse en el teléfono móvil (Android o iOS) como si fuera una app nativa, con icono en la pantalla de inicio, pantalla splash, y funcionamiento offline parcial. La PWA es la estrategia de móvil de VecinoClaro —no se planean apps nativas separadas en el corto plazo.

▸ Por qué PWA y no app nativa
Costo: una PWA es un solo código base que sirve para web + móvil. Una app nativa requiere mantener 2 codebases (iOS Swift + Android Kotlin) o usar React Native (complejidad adicional).

Adopción: los residentes no necesitan buscar "VecinoClaro" en App Store o Play Store. El admin les comparte un link, hacen "Agregar a pantalla de inicio", y ya tienen la app.

Updates instantáneos: cualquier mejora que deployemos está disponible para todos los usuarios al instante, sin pasar por revisión de App Store (que puede tardar días).

Almacenamiento: la PWA ocupa menos espacio que una app nativa, importante en Venezuela donde muchos usuarios tienen teléfonos con almacenamiento limitado.

Características PWA planeadas (Fase 11):

  • manifest.json con iconos adaptativos, nombre corto "VecinoClaro", colores de marca (emerald #047857), orientación portrait
  • Service Worker para cache offline: dashboard, avisos, y saldo accesibles sin conexión
  • Push notifications vía Web Push API para alertas críticas (pago aprobado, aviso urgente)
  • Install prompt personalizado con explicación de beneficios para residentes
  • Background sync para comprobantes: si el residente sube un comprobante sin conexión, se envía cuando vuelve el internet
